PDC Assignments: Circuit Satisfiability

Title:

Circuit Satisfiability

Topics:

Digital circuitry, parallel computing, performance analysis

Prerequisites:

Boolean logic, parallel loops, reduction, metering, speedup, computational efficiency

Instructions:

For a given binary circuit, the Circuit Satisfiability Problem is to find and output the circuit's "solutions"--all the inputs that cause that circuit to produce the value 1 (or true) (as opposed to 0 or false)--as well as a count of the total number of solutions. For example, here is a 32-bit circuit diagram:

The 32 vertical lines at the top are the binary inputs to the circuit; the single vertical line at the bottom-middle is the circuit's binary output. The circuit between the inputs and output contains AND-gates, OR-gates, and inverters, we can model that circuit in software using the boolean AND (&&), OR (||), and NOT (!) operators.

This problem can be solved using a brute-force approach, in which a for loop iterates through all the possible inputs, and if a given input i causes the circuit to produce 1, the program outputs i and increments a counter. After the for loop has finished, the program outputs the value of the counter.

Note: To find all solutions, the brute force algorithm must check 232 possible inputs to the circuit. It requires several minutes to finish on a typical Linux workstation (and many more minutes if using Python).

The assignment is:

  1. Starting with one of the sequential programs provided below, add internal timing calls (e.g., using MPI or OpenMP's Wtime functions) to the beginning and end of the sequential program to measure precisely how long it takes to solve the problem. Record that time in a spreadsheet, indicating that it is the time to solve the problem using 1 processing element (PE).
  2. Use parallel techniques to convert the sequential program into a parallel program that solves the problem. (Hint: Convert the for loop into a parallel loop.)
  3. In your spreadsheet, record the raw times your parallel solution requires to solve the problem using a given number of PEs. For each PE value, calculate the speedup and computational efficiency. Then create three line-charts that (respectively) show the relationship between the number of PEs and:
    1. the raw times,
    2. the speedup values, and
    3. the computational efficiency values.
    Depending on which software option you are using (see below), your instructor may provide additional details regarding their precise expectations regarding the number of PEs you are to use.
  4. Write a 1-page report in which you explain what your charts reveal about the relationships between the number of PEs and the raw time, the speedup, and the computational efficiency of your parallel program.

Deliverables:

Using the submission process at your institution, turn in:
  1. Your 1-page report.
  2. Your three charts (one per page, maximized to fill the page).
  3. Your spreadsheet, showing the data used to generate your charts.
  4. The source code of your parallel program.

Language Options:

C, C++, Java, or Python

Parallelization Options:

Different parallel software technologies can be used to speedup the solution to this problem, depending on the available parallel hardware environment. The following list presents the currently supported options; each link is to a folder containing a source program and a Makefile that can be used to build that program. Each source program solves the problem using a sequential brute-force approach, and can be used as starter code for this project.

  • On a multicore CPU (i.e., a shared-memory multiprocessor, such as a laptop, workstation, or even single-board computer), any of these may be used to solve the problem using CPU multithreading:
    • OpenMP (CPU): C or C++
    • Pthreads: C or C++
    • C++11 threads: C++
    • Java threads: Java
    • OpenACC (CPU): C or C++
  • On any multiprocessor (distributed-memory or shared-memory), any of these may be used to solve the problem using message passing:
    • MPI: C, C++, Java (requires OpenMPI), or Python (requires mpi4py)
  • On a GPU, any of these may be used to solve the problem using GPU multithreading:
    • CUDA: CUDA-C or CUDA-C++
    • OpenACC (GPU): C or C++
    • OpenMP (GPU): C or C++
