PDC Assignments: Circuit Satisfiability

Title: Circuit Satisfiability

Topics: digital circuitry, parallel computing

Prerequisites: Boolean logic, parallel loops, speedup, computational efficiency

Instructions:

For a given binary circuit, the Circuit Satisfiability Problem is to find the set of inputs that cause that circuit to produce 1 (as opposed to 0 as its output. For example, here is a 32-bit circuit diagram:

Since the circuit contains AND-gates, OR-gates, and inverters, we can model that circuit in software using the boolean AND (&&), OR (||), and NOT (!) operators. The sequential programs circuitSolver.c, circuitSolver.cpp, circuitSolver.java, and circuitSolver.py each take this approach, solving the problem using a brute-force approach. Each program uses a for loop to iterate through all possible inputs, outputting those inputs that cause the circuit to produce 1.

Note: To find all solutions, this sequential program must check all 232 possible inputs to the circuit. It requires several minutes to finish on a typical Linux workstation (and many minutes if using Python).

The assignment is to:

  1. Add timing calls to the sequential solution to measure precisely how long it takes to solve the problem. Record that time in a spreadsheet, indicating that it is the time to solve the problem using 1 processing element (PE).
  2. Use parallel techniques to create a parallel program that solves the problem. (Hint: Replace the for loop with a parallel loop.)
  3. In your spreadsheet, record the raw times your parallel solution requires to solve the problem using a given number of PEs. For each PE value, calculate the speedup and computational efficiency. Then create three line-charts that (respectively) show the relationship between the number of PEs and:
    1. the raw times,
    2. the speedup values, and
    3. the computational efficiency values.
    Your instructor will provide additional details regarding the precise expectations (which vary with different software options).
  4. Write a 1-page essay in which you explain what your charts reveal about the relationship between the number of PEs and the raw time, speedup, and computational efficiency of your parallel program.

Deliverables:

Using the submission process at your institution, turn in:
  1. The source code of your parallel program.
  2. Your spreadsheet, showing the raw data.
  3. Your three charts.
  4. A copy of your 1-page essay.

Software/Hardware Options:

  • Shared Memory (Multithreading, e.g. OpenMP)
  • Message-Passing (MPI)
  • GPU (CUDA, OpenACC, OpenMP)

Language Options: C, C++, Java, or Python
